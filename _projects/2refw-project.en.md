# SEO Challenge – 2REFW

Project completed as part of the **2REFW (WordPress and SEO)** module at **SUPINFO Lyon**, individually.

**Grade achieved: 91.38 / 100**

## Result

* **Website (now offline):** https://robin-massonnat.dev
* **Target keyword:** **Freelance Developer Lyon SUPINFO**

## 1. Project Objective

The objective of the challenge was to design an SEO-optimized WordPress website capable of ranking for a specific and
competitive local search query.

The main requirements were:

* use a CMS, in this case **WordPress**;
* apply SEO best practices;
* structure the website around a primary keyword;
* produce relevant and useful content;
* monitor ranking progress using SEO tools.

## 2. Initial Choices

### Domain Name

**Selected domain:** https://robin-massonnat.dev

This choice was made for several reasons:

* the `.dev` extension aligns with a technical profile;
* a simple, readable, and professional domain name;
* a clear identity using first and last name;
* availability at the time of registration;
* free of charge thanks to the student package.

### Hosting

**Hosting provider:** ElypseCloud, student Plesk web hosting plan.

This choice was made because of:

* its ease of administration;
* direct management through Plesk;
* good performance for a modest-sized website;
* no cost for students.

## 3. SEO Strategy

### 3.1 Primary Keyword

**Freelance Developer Lyon SUPINFO**

The content and optimization strategy was built around this query, with natural integration into:

* the `<title>`;
* the meta description;
* the homepage H1;
* editorial content;
* internal linking anchor texts;
* blog articles.

### 3.2 Search Intent

The query targets a commercial and local search intent. Users are generally looking for:

* a freelance service provider;
* a local developer;
* a credible and identifiable profile;
* a portfolio or proof of experience;
* a direct way to get in touch.

### 3.3 Competitor Analysis

The competition mainly consisted of:

* local freelancers;
* web agencies;
* intermediary platforms;
* profiles ranking for similar queries.

The chosen differentiation strategy focused on:

* a **personal profile**;
* **SUPINFO affiliation**;
* **Lyon-based location**;
* a **freelance-oriented approach**;
* **high-quality, well-structured content**.

### 3.4 Commercial Positioning

The selected positioning combines:

* technical expertise;
* academic credibility;
* geographical proximity;
* clear and straightforward communication;
* conversion-oriented content.

## 4. Website Architecture

The website was designed as a coherent structure built around a pillar page and supporting pages.

### 4.1 Main Pages

* **Home:** pillar page optimized for the primary keyword;
* **Services:** dedicated pages for web development, SaaS, and system administration;
* **Blog:** editorial content targeting long-tail keywords;
* **Contact:** conversion-focused page.

### 4.2 Internal Linking Strategy

The internal structure was designed to:

* guide users toward useful pages;
* reinforce topical relevance;
* distribute internal authority;
* facilitate search engine indexing.

Simplified diagram:

```txt
Home
  ↓
Services
  ↓
Blog
  ↓
Contact
```

## 5. Actions Implemented

### 5.1 Page Creation and Optimization

Each page was optimized with a clear structure:

* a unique H1;
* a relevant title;
* a carefully written meta description;
* properly structured headings;
* readable and targeted content;
* visible calls to action.

### 5.2 Landing Page

The homepage served as the project's central page.

It was designed to:

* clearly present the profile;
* include the target keyword;
* build trust through proof elements;
* direct users toward services and contact information;
* address the main search intent.

### 5.3 Service Pages

Dedicated pages were added to comply with the requirements and improve topical SEO.

Included services:

* custom web development;
* SaaS application development;
* system administration.

These pages help to:

* clarify the service offering;
* target complementary keywords;
* reinforce website consistency;
* improve internal linking.

### 5.4 Blog

The blog served as an editorial platform for publishing targeted content about:

* freelancing;
* local SEO;
* choosing a developer;
* pricing;
* maintenance;
* SUPINFO.

Role of the blog:

* target long-tail queries;
* enrich semantic relevance;
* generate fresh content;
* support the pillar page;
* create links toward commercial pages.

### 5.5 On-Page Optimizations

The optimizations focused on:

* page titles;
* meta descriptions;
* heading hierarchy;
* alt attributes;
* content readability;
* consistency of internal anchors.

### 5.6 Performance and Technical Improvements

Technical optimizations were also implemented:

* image compression and optimization;
* lazy loading of non-critical resources;
* improved loading times;
* clean structure to facilitate crawling.

### 5.7 Tools Used

The tools and services used for SEO monitoring included:

* **Google Search Console**;
* **Google Site Kit**;
* **Semrush**;
* **MathRank SEO**;
* **PageSpeed Insights**;
* **Cloudflare**;
* **Imagify**;
* **Autoptimize**;
* **Wordfence**;
* **WP Consent API**;
* **Otter Blocks**;
* **Neve FSE**.

## 6. Results Achieved

### 6.1 Rankings

The website achieved rapid results on the target query, with steady ranking improvements following content publication
and technical adjustments.

### 6.2 Traffic and Monitoring

The first organic visits were observed shortly after indexing.

However, these statistics should be interpreted with caution, as part of the traffic originated from:

* classroom testing;
* personal checks;
* bots and crawlers;
* repeated re-indexing.

### 6.3 Project Evolution

The strategy evolved over time:

* homepage optimized from the beginning;
* service pages added after reviewing the project requirements;
* progressive publication of blog articles;
* continuous SEO adjustments;
* resolution of occasional technical issues.

## 7. Limitations Encountered

The project also revealed several limitations:

* still relatively low domain authority;
* limited number of backlinks;
* active competition on the target query;
* traffic data partially skewed by testing activities;
* constraints related to certain SEO tools.

These limitations did not prevent the project from achieving strong performance, but they remain areas for improvement.

## 8. What I Learned

This project allowed me to gain a better understanding of:

* the logic behind an SEO-oriented website;
* the importance of semantic consistency;
* the role of internal linking;
* the value of a clear and scalable structure;
* the difference between technical optimization and editorial optimization;
* the real impact of consistently publishing content.

## 9. Conclusion

This 2REFW project demonstrates that a well-structured, consistent, and properly maintained WordPress website can
achieve strong SEO results on a targeted local query.

The most decisive factors were:

* selecting a precise keyword;
* creating a solid landing page;
* adding dedicated service pages;
* publishing relevant blog articles;
* regularly monitoring performance;
* implementing technical fixes quickly.

Ultimately, this challenge provided an excellent opportunity to put SEO fundamentals into practice through a technical,
editorial, and strategic approach.
