# SEO Challenge – 2REFW

Projet réalisé dans le cadre du module **2REFW (WordPress et SEO)** à **SUPINFO Lyon**, en solo.

**Note obtenue : 91,38 / 100**

## Résultat

- **Site (désormais éteint)** : [https://robin-massonnat.dev](https://robin-massonnat.dev)
- **Mot-clé cible** : **Développeur Freelance Lyon SUPINFO**

## 1. Objectif du projet

L’objectif du challenge était de concevoir un site WordPress optimisé SEO, capable de se positionner sur une requête
locale précise et concurrentielle.

Les attentes principales étaient les suivantes :

- utiliser un CMS, ici **WordPress** ;
- appliquer les bonnes pratiques de référencement naturel ;
- structurer le site autour d’un mot-clé principal ;
- produire des contenus cohérents et utiles ;
- suivre l’évolution du positionnement à l’aide d’outils SEO.

## 2. Choix de départ

### Domaine

**Domaine retenu :** [https://robin-massonnat.dev](https://robin-massonnat.dev)

Ce choix s’explique par plusieurs raisons :

- extension `.dev` cohérente avec un profil technique ;
- domaine simple, lisible et professionnel ;
- identité claire avec prénom et nom ;
- disponibilité au moment de l’enregistrement ;
- coût nul grâce au pack étudiant.

### Hébergement

**Hébergement utilisé :** ElypseCloud, offre web Plesk étudiante.

Ce choix a été retenu pour :

- sa simplicité d’administration ;
- la gestion directe via Plesk ;
- un bon niveau de performance pour un site de taille modeste ;
- son coût nul pour les étudiants.

## 3. Stratégie SEO

### 3.1 Mot-clé principal

**Développeur freelance Lyon SUPINFO**

La stratégie de contenu et d’optimisation a été construite autour de cette requête, avec intégration naturelle dans :

- le `<title>` ;
- la meta description ;
- le H1 de la page d’accueil ;
- le contenu éditorial ;
- les ancres de maillage interne ;
- les articles de blog.

### 3.2 Intention de recherche

La requête cible une intention commerciale et locale. L’utilisateur cherche généralement :

- un prestataire freelance ;
- un développeur local ;
- un profil crédible et identifiable ;
- un portfolio ou des preuves de réalisation ;
- un moyen de contact direct.

### 3.3 Analyse de la concurrence

La concurrence se compose principalement :

- de freelances locaux ;
- d’agences web ;
- de plateformes intermédiaires ;
- de profils positionnés sur des requêtes proches.

L’opportunité retenue était de se différencier sur l’axe suivant :

- **profil personnel** ;
- **ancrage SUPINFO** ;
- **localisation Lyon** ;
- **approche freelance** ;
- **contenu de qualité et structuré**.

### 3.4 Positionnement commercial

Le positionnement choisi combine :

- expertise technique ;
- crédibilité académique ;
- proximité géographique ;
- discours clair et direct ;
- contenus pensés pour la conversion.

## 4. Architecture du site

Le site a été pensé comme un ensemble cohérent avec une page pilier et des pages satellites.

### 4.1 Pages principales

- **Accueil** : page pilier optimisée sur le mot-clé principal ;
- **Services** : pages dédiées au développement web, au SaaS et à l’administration système ;
- **Blog** : contenu éditorial pour travailler les longue traîne ;
- **Contact** : page de conversion.

### 4.2 Logique de maillage interne

La structure interne a été conçue pour :

- guider l’utilisateur vers les pages utiles ;
- renforcer la pertinence thématique ;
- distribuer l’autorité interne ;
- faciliter l’indexation par les moteurs de recherche.

Schéma simplifié :

```txt
Accueil
  ↓
Services
  ↓
Blog
  ↓
Contact
```

## 5. Actions réalisées

### 5.1 Création et optimisation des pages

Chaque page a été optimisée avec une structure claire :

- un H1 unique ;
- un title pertinent ;
- une meta description travaillée ;
- des sous-titres hiérarchisés ;
- des contenus lisibles et ciblés ;
- des appels à l’action visibles.

### 5.2 Landing page

La page d’accueil a servi de page centrale du projet.

Elle a été construite pour :

- présenter clairement le profil ;
- reprendre le mot-clé cible ;
- rassurer avec des éléments de preuve ;
- orienter vers les services et le contact ;
- répondre à l’intention de recherche principale.

### 5.3 Pages services

Des pages dédiées ont été ajoutées pour respecter les consignes et améliorer le SEO thématique.

Services présents :

- développement web sur mesure ;
- application SaaS ;
- administration système.

Ces pages permettent de :

- clarifier l’offre ;
- travailler des mots-clés complémentaires ;
- renforcer la cohérence du site ;
- améliorer le maillage interne.

### 5.4 Blog

Le blog a servi de support éditorial pour publier des contenus ciblés autour de :

- freelancing ;
- SEO local ;
- choix d’un développeur ;
- tarifs ;
- maintenance ;
- SUPINFO.

Rôle du blog :

- viser des requêtes longue traîne ;
- enrichir le champ sémantique ;
- générer du contenu frais ;
- soutenir la page pilier ;
- créer des passerelles vers les pages commerciales.

### 5.5 Optimisations on-page

Les optimisations appliquées ont porté sur :

- les titres de pages ;
- les méta-descriptions ;
- la hiérarchie des titres ;
- les balises alt ;
- la lisibilité du contenu ;
- la cohérence des ancres internes.

### 5.6 Performance et technique

Des optimisations techniques ont également été mises en place :

- compression et optimisation des images ;
- chargement différé des ressources non critiques ;
- amélioration du temps de chargement ;
- structure propre pour favoriser le crawl.

### 5.7 Outils utilisés

Les outils et services mobilisés pour le suivi SEO ont été :

- **Google Search Console** ;
- **Google Site Kit** ;
- **Semrush** ;
- **MathRank SEO** ;
- **PageSpeed Insights** ;
- **Cloudflare** ;
- **Imagify** ;
- **Autoptimize** ;
- **Wordfence** ;
- **WP Consent API** ;
- **Otter Blocks** ;
- **Neve FSE**.

## 6. Résultats obtenus

### 6.1 Positionnement

Le site a obtenu des résultats rapides sur la requête cible, avec une progression régulière du positionnement après
publication des contenus et ajustements techniques.

### 6.2 Trafic et suivi

Les premières visites issues de la recherche ont été observées rapidement après indexation.

Cependant, les statistiques doivent être interprétées avec prudence, car une partie du trafic provient :

- des tests de la classe ;
- des vérifications personnelles ;
- des robots et crawlers ;
- des réindexations successives.

### 6.3 Évolution du projet

La stratégie a évolué au fil du temps :

- page d’accueil optimisée dès le départ ;
- ajout de pages services après relecture des consignes ;
- publication progressive d’articles de blog ;
- ajustements SEO continus ;
- correction de problèmes techniques ponctuels.

## 7. Limites rencontrées

Le projet a aussi montré plusieurs limites :

- autorité du domaine encore faible ;
- volume de backlinks limité ;
- concurrence active sur la requête cible ;
- données de trafic partiellement faussées par les tests ;
- contraintes liées à certains outils SEO.

Ces limites n’ont pas empêché le projet d’atteindre un bon niveau de performance, mais elles restent des axes
d’amélioration.

## 8. Ce que j’ai appris

Ce projet m’a permis de mieux comprendre :

- la logique d’un site pensé pour le référencement ;
- l’importance de la cohérence sémantique ;
- le rôle du maillage interne ;
- l’intérêt d’une structure claire et évolutive ;
- la différence entre optimisation technique et optimisation éditoriale ;
- l’impact réel de la régularité dans la publication de contenu.

## 9. Conclusion

Ce projet 2REFW montre qu’un site WordPress bien structuré, cohérent et suivi dans le temps peut obtenir de bons
résultats SEO sur une requête locale ciblée.

Les points les plus déterminants ont été :

- le choix d’un mot-clé précis ;
- la création d’une landing page solide ;
- l’ajout de pages services dédiées ;
- la publication d’articles de blog pertinents ;
- le suivi régulier des performances ;
- les corrections techniques rapides.

Au final, ce challenge a été une bonne mise en pratique des bases du SEO, avec une approche à la fois technique,
éditoriale et stratégique.
